# Вариант 4 — `POST /v1/match/analyze`

## Что делает

1. Как **вариант 2** — семантический поиск по **проиндексированным** лотам в pgvector: кто из сохранённых ближе к профилю.
2. Затем **OpenAI** получает профиль, найденные **`snippets`** (и опционально строку **`incoming`** с карточкой лота с вашего API) и формирует **`summary`**, **`verdicts`** по элементам топа.

## Что в ответе

- **Структурированный разбор** (вердикты по топ-лотам, сводка).
- **`matches`** — как у варианта 2: `lot_id`, `score`, `snippets` (поисковая часть сохраняется в ответе).

## Требования

- Лоты в топе должны быть в БД: [`POST /v1/lots/{lot_id}/index`](indexing.md).
- **`OPENAI_API_KEY`** в `.env`.
- Профиль — в теле или `.env`.

## Важно про `lot_id`

В запросе **не** передаётся «проанализируй только лот N». Вы задаёте, например, **`top_lots`**; **какие именно лоты** в топе — смотрите в ответе в полях **`lot_id`** у `matches` / `verdicts` (те же id, что при индексации).

Разбор **ровно одного** лота по тексту без опоры на топ базы — [вариант 3 — `/v1/lot/analyze`](lot-analyze.md).

## Когда использовать

Дашборды и списки: «по **всей нашей** проиндексированной базе кто в топе и **почему словами**».

## Сводка по режимам

| База (индекс) | OpenAI | Один лот текстом в запросе |
|:--:|:--:|:--:|
| да | да | опционально (`incoming`) |

## Примеры `curl`

Базовый запрос:

```bash
curl -s --max-time 300 -X POST http://127.0.0.1:8083/v1/match/analyze \
  -H "Content-Type: application/json" \
  -d '{"top_lots":5,"snippets_per_lot":2}'
```

С дополнительным контекстом лота (опционально):

```bash
curl -s --max-time 300 -X POST http://127.0.0.1:8083/v1/match/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "top_lots": 5,
    "snippets_per_lot": 2,
    "incoming": "{\"id\":50225530,\"title\":\"...\",\"description\":\"...\"}"
  }'
```

Первый запрос после перезапуска API может быть долгим — не уменьшайте `--max-time` без нужды.
