# Вариант 3 — `POST /v1/lot/analyze`

## Что делает

- **Только OpenAI.** Таблица с чанками для **поиска** в этом запросе **не** используется: лот **не обязан** быть в индексе.
- В теле передаётся один большой текст **`lot_text`** (+ опционально **`profile`**, иначе из `.env`).
- Модель возвращает **человекочитаемый разбор** одного лота.

## Что в ответе

- **`summary`** — краткое содержание.
- **`fit`** — ожидаемые значения: `подходит` | `сомнительно` | `не подходит` (строка от модели).
- **`reason`** — обоснование.
- **`checks`** — дополнительный текст проверок от модели или `null`.

## Требования

- **`OPENAI_API_KEY`** в `.env` (или в окружении контейнера).
- Профиль компании — в теле или `.env`.

## Когда использовать

**Карточка одного тендера:** пользователь открыл конкретный лот — бэкенд собирает текст (`title`, описание, важные поля) и запрашивает вердикт **без** предварительной загрузки лота в RAG.

## Сводка по режимам

| База (индекс) | OpenAI | Один лот текстом в запросе |
|:--:|:--:|:--:|
| нет | да | да (`lot_text`) |

## Пример `curl`

```bash
curl -s --max-time 120 -X POST http://127.0.0.1:8083/v1/lot/analyze \
  -H "Content-Type: application/json" \
  -d '{"lot_text":"Услуги хостинга, продление. Астана, сумма 50000 тг."}'
```

Опционально позже для контекста «похожие ТЗ из нашей базы» можно дополнительно вызывать [вариант 4](match-analyze.md) с **`incoming`**, но основной сценарий экрана одного лота — этот эндпоинт.
